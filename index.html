Git Essential basic commands

What is Git and Github?

GitHub Is a social media for programmer, which can also be use to edit and make changes to file from around the world base on a cloud-base hosting platform. Git is an open-source coding management system for tracking changes in any set of files when working in a team, this help to assign work among programmers

How to install Git and GitHub?

Firstly, you need to create a GitHub account by following this link: Create your GitHub account
Next login to your account and create a repositor
Next install Git in this link Install GitHub

Git basic commands

1. Git config

Git config allow users to set up a username / email address for main configuration file.
How to: git config --global user.name = “” and git config --global user. Email = “”

2. Git clone

Git clone allow users to clone a repository into a new directory. How to: click “clone repository” then input your URL

3. Git status

How to: git status

4. Git add

Git add allow users to add new or change files from your local machine to your GitHub repository How to: git add *for specific file git add . *to all files

5. Git commit

Git commit let users to snapshot change stages on repository file. How to: git commit -m “”

6. Git pull

Git pull is used to download content from your Github repository to your local machine. How to: git pull

Resoling Conflict

Conflict can occur when a new file is being create in local computer and push to Git Hub which also have a file with a same name but different content. This is how to solve the conflict:
1. Another team member create a new file in your repository. E.g: texting.txt
2. The content in testing.txt is “online”
3. A new file with the same “online.txt” name was create in your local machine without knowing there a same file on your repository
4. After that you using git command “git add .”, ”git commit”, ”git push” to push your file online. However, the terminal will stop you and force you to fix the file.
5. After fixing the file, using “git add .”, ‘git commit” and “git push” again.
6. Conflict resolve and content is update.

Branching

Git branch is basically to help you to separate your file into different category and doesn’t affect each other. This allows you to develop features, fix bugs, or safely experiment with new ideas in a contained area of your repository.
